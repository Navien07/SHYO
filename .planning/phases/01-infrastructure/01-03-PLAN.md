  <action>
Create `.github/dependabot.yml` at the repository root (the `.github/` directory must be at the project root, not inside `src/`):

```yaml
# .github/dependabot.yml
# Source: https://docs.github.com/en/code-security/dependabot/dependabot-version-updates/configuring-dependabot-version-updates
version: 2
updates:
  - package-ecosystem: "npm"
    directory: "/"
    schedule:
      interval: "weekly"
      day: "monday"
      time: "09:00"
      timezone: "Asia/Kuala_Lumpur"
    open-pull-requests-limit: 5
    labels:
      - "dependencies"
    groups:
      sanity:
        patterns:
          - "sanity"
          - "@sanity/*"
          - "next-sanity"
      next-intl:
        patterns:
          - "next-intl"
      tailwind:
        patterns:
          - "tailwindcss"
          - "@tailwindcss/*"
```

Commit and push:
```bash
git add .github/dependabot.yml
git commit -m "chore: add dependabot weekly dependency update schedule"
git push origin master
```

Dependabot activates automatically once `.github/dependabot.yml` is on the default branch. No dashboard configuration is required — GitHub reads this file.
