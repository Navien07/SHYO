## Spacing Scale

Declared values (multiples of 4 — Tailwind v4 default scale applies):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding |
| sm | 8px | Card inner padding (compact), gap between filter pills |
| md | 16px | Default element spacing, card content padding |
| lg | 24px | Section internal padding, grid gap |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page section breaks (Our Team tier separators) |
| 3xl | 64px | Top-of-page vertical padding |

Exceptions:
- Touch targets for filter buttons: minimum 44px height (BRAND-04 requirement, WCAG 2.1 AA)
- Team photo avatars: 80px square (president tier), 64px square (senior/committee tiers)

Font weights used: 400 (regular) and 600 (semibold) — no others.

## Page-Level Visual Contracts

### Our Team (`/[locale]/team`)

**Layout:**
- Page title (h1, Display 28px/600): "Our Team" (translated via `t('team.title')`)
- Three visual tiers rendered top-to-bottom, each separated by 48px vertical gap
- **President tier:** Single card, centred, max-width 320px, slightly larger photo (80px square avatar or full-width image at aspect-ratio 1:1, max 320px wide)
- **Senior Officers tier:** Horizontal row, `flex flex-wrap gap-6 justify-center`, cards max 200px wide each
- **Committee Members tier:** Responsive grid — 2 columns on mobile (sm), 3 columns on tablet (md), 4 columns on desktop (lg)

**Card anatomy:**
- Photo: `aspect-square object-cover rounded-full` (circular crop) — 80px (president), 64px (senior/committee)
- Photo placeholder (no image uploaded): initials avatar — 2-letter monogram, bg-brand-green text-white, same dimensions
- Name: Heading 20px/600, text-foreground
- Role: Label 14px/400, text-muted-foreground
- Card container: `bg-card rounded-lg p-4 text-center shadow-sm border border-border`
- Within each tier: cards ordered by `order` field ascending; members with no `tier` fall into committee tier

**Tier section headers:**
- Left-border accent: `border-l-4 border-brand-green pl-3` applied to `<h2>`
- Text: Heading 20px/600, text-foreground
- Keys: `t('team.presidentSection')`, `t('team.seniorSection')`, `t('team.committeeSection')`

**Empty state (no members at all):**
- Centred block: icon (`Users` from lucide-react, 48px, text-muted-foreground) + heading "Team details coming soon" + no body text
- Source: CONTEXT.md locked decision

### PDF Document Library (`/[locale]/documents`)

**Layout:**
- Page title (h1, Display 28px/600): "Documents" (`t('documents.title')`)
- Filter bar: `flex flex-wrap gap-2 mb-6` row of shadcn Buttons
  - Active: `variant="default"` with `bg-brand-green text-white` override
  - Inactive: `variant="outline"`
  - Minimum height 44px (touch target requirement)
  - Labels: "All", "Annual Report", "Constitution", "Minutes", "Policy", "Other" (`t('documents.filterAll')` + raw category names)
- Document display: `<table>` with responsive wrapper `overflow-x-auto`
  - Columns: Title | Category | Year | Upload Date | File Size | Download
  - Table header: `bg-muted text-muted-foreground text-xs uppercase tracking-wide font-medium`
  - Table rows: `border-b border-border hover:bg-muted/50 transition-colors`
  - Title cell: Body 16px/400, text-foreground
  - Metadata cells (category, year, date, size): Label 14px/400, text-muted-foreground
  - Download cell: shadcn Button variant="outline" size="sm" with `Download` icon from lucide-react + `t('documents.download')` label; `target="_blank" rel="noopener noreferrer"`
  - File size: formatted as "X.X MB" / "X.X KB" via `formatBytes()` helper
  - Upload date: formatted as locale date string (`toLocaleDateString()`)

**Empty state (no documents):**
- Centred: `FileText` icon (48px, text-muted-foreground) + "No documents available yet"
- Source: CONTEXT.md locked decision

**Empty filter result (filter active but no matches):**
- Centred in table body area: "No {category} documents found." — constructed from active filter label

## Interaction Contracts

### Document Filter

- Default state: "All" tab active, all documents visible
- Click any category tab: sets active category, filters instantly (no animation, no debounce)
- Active tab: `variant="default"` with brand-green fill; inactive: `variant="outline"`
- No URL change, no scroll-to-top — pure in-memory filter
- Source: CONTEXT.md locked decision (client-side, no URL param)

### Programme Card — Read More

- Entire card is NOT a link; only the "Read More" button navigates
- Hover state on card: subtle `shadow-md` upgrade (from `shadow-sm`)
- Button uses `Link` from `@/i18n/navigation` (not `next/link`) — locale-aware routing (Phase 2 decision)

### Document Download

- Opens PDF in new tab: `target="_blank" rel="noopener noreferrer"`
- No confirmation dialog — direct navigation
- Icon: `Download` from lucide-react, 16px, inline with button text

### Team Photo Fallback

- If `photo` is null/undefined: render initials avatar
- Initials: first letter of first word + first letter of last word from name (active locale)
- Avatar: circular, bg-brand-green, text-white, same dimensions as photo would be
- If name has only one word: use first two characters

## Accessibility Notes

- All filter buttons: min-height 44px (BRAND-04, WCAG 2.1 AA touch target)
- Team photo `<img>`: `alt={member.name[locale]}` — member name in active locale
- Programme hero image: `alt={programme.title[locale]}`
- Download button: `aria-label={t('documents.download') + ' ' + doc.title[locale]}` to disambiguate multiple "Download" buttons in screen readers
- Category badges: purely decorative — no additional aria role needed; text is already visible
- Tier headings (`<h2>`) maintain proper heading hierarchy below page `<h1>`
- PortableText renders semantic HTML (`<h2>`, `<h3>`, `<p>`, `<ul>`) — no additional aria required
- Colour contrast: brand-green (#32a852) on white bg ≈ 3.8:1 — use as background with white text or text on light bg with increased font-weight; for category badge text use `text-brand-green-dark` (#1e7a36) on white for ≥ 4.5:1
