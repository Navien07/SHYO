## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon gaps, checkbox-to-label spacing |
| sm | 8px | `gap-2` / `mb-2` | Compact field spacing, label-to-input gap |
| md | 16px | `gap-4` / `px-4` | Default element spacing, form field stacking (`space-y-4`) |
| lg | 24px | `gap-6` / `py-6` | Section padding inside form card |
| xl | 32px | `gap-8` / `py-8` | Layout gaps between form and page sections |
| 2xl | 48px | `py-12` | Footer section padding (established in Phase 2) |
| 3xl | 64px | `py-16` | Page-level top/bottom padding (`py-16` on `<main>`) |

Exceptions:
- Touch targets minimum 44px height (`min-h-[44px]`) — all interactive form elements, PDPA checkbox label, interest checkboxes. This is a WCAG 2.1 AA requirement carried from Phase 2 (BRAND-04).
- Form submit button: full width on mobile (`w-full`), fixed height 48px (`h-12`) for prominence.
- **NAMED EXCEPTION — success modal vertical padding:** `py-10` (40px) is used on the success modal card (`bg-white rounded-2xl shadow-xl px-8 py-10 max-w-sm`). This matches `ContactForm.tsx` line 124 exactly. Changing to `py-12` (48px) would create visual inconsistency between the contact success state and the membership success state — both modals must look identical. This value is intentionally preserved and must not be changed to align the two components' appearance.

## Typography

Source: globals.css + existing page patterns.

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|----------|
| Body / Field text | 16px | 400 (regular) | 1.5 | `text-base font-normal` |
| Label | 14px | 400 (regular) | 1.4 | `text-sm font-normal` |
| Heading (page H1) | 36px | 700 (bold) | 1.2 | `text-4xl font-bold` |
| Sub-heading (section H2) | 20px | 400 (regular) | 1.3 | `text-xl font-normal` |

Tamil locale override: line-height 1.7 (set globally on `html[lang="ta"]` in globals.css — no additional class needed).

**Weights in use: 400 regular (body, labels, sub-headings) and 700 bold (page H1 only).** Executor must not introduce additional weights.

Labels are visually differentiated from body text by size (14px vs 16px), not by weight. Sub-headings are visually differentiated from body text by size (20px vs 16px), not by weight.

Font size range: 14px (labels, error text) → 16px (body/inputs) → 20px (section headers) → 36px (page H1). No other sizes.

Error text: 14px, weight 400, color `text-red-600` — matches existing ContactForm pattern.

**NAMED EXCEPTION — `font-medium` in ContactForm.tsx:** The existing `ContactForm.tsx` uses `font-medium` (weight 500) on `<label>` elements at lines 49, 60, 71, and 82. This is a codebase artifact from Phase 3 — it was not introduced by Phase 5. The MembershipForm.tsx must use `font-normal` (weight 400) for its labels per this contract. The existing ContactForm.tsx labels are not modified in Phase 5 (out of scope). This named exception exists solely to document the discrepancy; it does not grant permission to use `font-medium` in any Phase 5 new files.

## Color

Source: globals.css brand token block + existing component patterns.

| Role | Value | CSS Var / Class | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | `oklch(1 0 0)` = white | `--background` / `bg-background` | Page background, form field backgrounds, modal background |
| Secondary (30%) | `oklch(0.97 0 0)` ≈ off-white | `--secondary` / `bg-secondary` | Card surfaces, muted section backgrounds (e.g. privacy policy section blocks) |
| Brand Green (accent primary) | `#32a852` | `--color-brand-green` / `bg-brand-green` | Submit button background, focus rings (`focus:ring-brand-green`), success icon, links |
| Brand Green Dark (accent hover) | `#1e7a36` | `--color-brand-green-dark` / `bg-brand-green/90` | Hover state on brand-green buttons (use `hover:bg-brand-green/90` per ContactForm) |
| Brand Saffron (accent secondary) | `#c96a00` | `--color-brand-saffron` | Homepage "Be Part of Us" CTA button only — not used in membership form itself |
| Brand Charcoal | `#1a1a1a` | `--color-brand-charcoal` | Footer background only |
| Destructive | `oklch(0.577 0.245 27.325)` ≈ red-600 | `--destructive` | Form send error banner border; inline field error text (`text-red-600`) |
| Muted text | `oklch(0.556 0 0)` ≈ gray-500 | `--muted-foreground` | Helper text, placeholder text, success modal body copy |

Accent (brand-green) reserved for:
1. Primary submit button (`bg-brand-green` with `text-white`)
2. Input focus rings (`focus:ring-2 focus:ring-brand-green`)
3. Success checkmark icon and success modal button
4. Inline text links (hover:underline links in Privacy Policy page)
5. PDPA consent checkbox — checked state indicator

Brand saffron is reserved exclusively for the homepage "Be Part of Us" CTA (established Phase 3). Do not use saffron inside the membership form itself.

Destructive / error color: `text-red-600` for field-level error text; `bg-red-50 border border-red-200 text-red-700` for the global send error banner — matches ContactForm.tsx pattern exactly.

## Layout Contract

### Membership Form Page (`/[locale]/membership`)

```
<main className="max-w-2xl mx-auto px-4 py-16">
  <h1> {t('title')} — "Be Part of Us"         36px bold
  <p>  Optional subtitle / description         16px regular, text-muted-foreground, mb-8

  <MembershipForm>                             card or flat (no card border — consistent with contact page)
    [Row 1: Full Name — full width]
    [Row 2: DOB — full width on mobile, 1/2 width on desktop | Gender — 1/2 width on desktop]
    [Row 3: Mobile — full width on mobile, 1/2 width on desktop | Email — 1/2 width on desktop]
    [Row 4: Area / Location — full width]
    [Row 5: Interests — 2-col checkbox grid on desktop, 1-col on mobile]
    [Row 6: PDPA consent — full width checkbox with paragraph label]
    [Row 7: Submit button — full width]
```

Desktop 2-column pairs: DOB + Gender, Mobile + Email. Use `grid grid-cols-1 sm:grid-cols-2 gap-4` for those rows.

Full-width rows: Full Name, Area/Location, Interests, PDPA, Submit.

Form max-width: `max-w-2xl` (672px) — narrower than contact page's `max-w-6xl` because this is a form-only page with no sidebar column.

### Privacy Policy Page (`/[locale]/privacy`)

```
<main className="max-w-3xl mx-auto px-4 py-16">
  <h1>  Privacy Policy                         36px bold, mb-8
  <p>   Last updated / effective date          14px, text-muted-foreground, mb-12

  <section> × 5  (one per PDPA section)
    <h2> Section heading                       20px regular, mb-3
    <p>  Section body                          16px regular, mb-8, leading-relaxed
```

No card backgrounds — plain prose layout.

### Admin Guide Page (`/[locale]/admin-guide`)

```
<main className="max-w-3xl mx-auto px-4 py-16">
  <h1>  Admin Guide                            36px bold, mb-2
  <p>   Subtitle                               16px, text-muted-foreground, mb-12

  <section> × 6  (one per guide section, numbered)
    <h2> Step N: [Title]                       20px regular, mb-3

```

Use `<meta name="robots" content="noindex, nofollow">` via `generateMetadata` — omit from sitemap.ts entirely (cleanest approach per RESEARCH.md Pattern 4 decision).

## Interaction Contract

### Form Submission States

| State | Visual Indicator |
|-------|-----------------|
| Idle | Submit button enabled (if PDPA checked) or disabled (if PDPA unchecked) |
| Pending | Submit button shows `...` text, `disabled={pending}` — matches ContactForm.tsx line 93 |
| Field error | Inline `<p>` under each failing field, `text-red-600 text-sm`, `aria-describedby` on input |
| Send error | Red banner above form (`bg-red-50 border-red-200 text-red-700`) — matches ContactForm.tsx lines 39-46 |
| Success | Modal overlay appears (`fixed inset-0 z-50`), form stays beneath, close button remounts form |

### PDPA Checkbox Interaction

- Submit button is `disabled={pending || !pdpaChecked}` — disabled visually (shadcn Button handles `disabled:opacity-50 cursor-not-allowed`) when PDPA unchecked.
- Checking PDPA enables the button — no submit until consent given.
- Server-side: `z.literal(true)` rejects if PDPA not `true`. Error message `errorPdpaRequired` rendered inline under the checkbox.

### Success Modal

- Matches ContactForm.tsx implementation exactly:
  - Fixed overlay with `bg-black/40` backdrop
  - White card `rounded-2xl shadow-xl px-8 py-10 max-w-sm` (see spacing exception above)
  - Brand-green checkmark icon in `w-14 h-14` circle with `bg-brand-green/10`
  - `h3` title at `text-xl font-normal`
  - `p` body copy at default size, `text-gray-500`
  - Close button: shadcn `<Button>` with `bg-brand-green hover:bg-brand-green/90 text-white px-8`, label: **"Done"**
  - Clicking backdrop also closes

### Focus Management

- All inputs: `focus:outline-none focus:ring-2 focus:ring-brand-green` — consistent with ContactForm.
- Tab order follows visual DOM order: Full Name → DOB → Gender → Mobile → Email → Area → Interests checkboxes (top-left, top-right, bottom-left, bottom-right, last) → PDPA → Submit.

## Accessibility Contract

Carried from Phase 2 (BRAND-09) and enforced in Phase 5:

| Requirement | Implementation |
|-------------|---------------|

| Alt text | No images in membership form; Privacy Policy and Admin Guide are text-only pages |
